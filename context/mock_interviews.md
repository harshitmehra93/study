# Mock Interview Tracker

Use this file to track timed mock-interview performance. The goal is to identify repeated execution gaps across modeling, correctness, implementation, communication, and time management—not merely to count completed mocks.

## Calibration

- `✅ Independent pass` = correct model and implementation within the round, no major hints, edge cases and complexity handled, and explanation was interview-ready.
- `🟡 Review` = partial solution, substantial correction, incomplete implementation, or important feedback still unresolved.
- `Uncalibrated` = outcome is known, but hint level, correctness review, or interviewer feedback has not been captured.

## Capture After Each Mock

Record only the information that changes training decisions:

1. Exact problem or design prompt.
2. Time to select the model and time to finish.
3. Hints or interviewer intervention.
4. Correctness, missed edge cases, and final complexity.
5. Communication and debugging quality.
6. First meaningful failure point.
7. One follow-up action and, when appropriate, a redo date.
